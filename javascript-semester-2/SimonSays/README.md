# Проект Simon Says — Описание Архитектуры и Реализации

Данный проект представляет собой классическую игру на развитие памяти "Simon Says", реализованную на чистом JavaScript (Vanilla JS). Архитектура приложения построена с использованием современных стандартов ECMAScript, объектно-ориентированного программирования (ООП) и асинхронного программирования. Код строго инкапсулирован, использует приватные поля и конечный автомат (State Machine) для управления логикой игры. Данная документация предназначена для ревью и объясняет выбор паттернов проектирования.

## 🏗 Архитектура и основные концепции

Основная логика инкапсулирована в рамках класса `Game`. В проекте сознательно применен подход единого контроллера состояния, который исключает неконтролируемые мутации и состояния гонки (Race conditions) при асинхронном взаимодействии. Модульность обеспечивается разделением логики состояния, рендеринга и работы с персистентным хранилищем.

### 1. Управление состоянием (Конечный автомат / State Machine)
Вместо сложных цепочек флагов (например, `isPlaying`, `isWaiting`), в приложении реализован паттерн "Конечный автомат" с помощью неизменяемого объекта `Status` (используется `Object.freeze`):
- `IDLE` — ожидание старта игры.
- `SHOWING_SEQUENCE` — этап демонстрации последовательности (взаимодействие с UI заблокировано).
- `WAITING_INPUT` — этап ожидания ввода пользователя.
- `GAME_OVER` — состояние проигрыша.

Переходы между состояниями строго контролируются через сеттер `status`, который дополнительно вызывает метод `updateControls()` для синхронизации интерфейса (блокировка/разблокировка кнопок) с текущим состоянием.

### 2. Инкапсуляция и ООП
Класс `Game` активно использует приватные поля (Private Class Features: `#status`, `#sequence`, `#currentStep`, `#record` и др.), что обеспечивает надежную защиту внутреннего состояния от случайных изменений извне. 
Доступ на чтение к необходимым параметрам осуществляется исключительно через публичные геттеры (например, `get status()`, `get cntOfBtn()`).

### 3. Асинхронное программирование (Async / Await)
Игра "Simon Says" базируется на временных задержках (показ карточек, обратный отсчет). В проекте реализован элегантный неблокирующий подход (Non-blocking I/O) с помощью `Promises` и синтаксиса `async/await`. 
- Утилитный метод `delay(ms)` используется в качестве Promisified-таймера. Это позволяет писать линейный, синхронно выглядящий и читаемый асинхронный код (например, циклы `for...of` в `playSequence()` и `RSG()`) без эффекта "Callback Hell" или сложно отслеживаемых `setTimeout`.

### 4. Паттерн Event Delegation (Делегирование событий)
Для обработки кликов по игровым элементам (карточкам) применён паттерн делегирования событий. Вместо того чтобы навешивать слушатели на каждую отдельную карточку (что негативно влияет на производительность и потребление памяти), используется единый слушатель на родительском контейнере (`#container`). Метод `e.target.closest('.card')` позволяет точно и безопасно определить интерактивный элемент, независимо от динамического создания карточек в DOM.

### 5. Персистентность данных (Local Storage API)
Состояние игры (текущий рекорд, счет, последовательность и статус) надежно сохраняется в браузере при любых изменениях состояний (методы `save()` и `load()`). Если пользователь случайно перезагрузит страницу во время игры, состояние корректно восстановится прямо с того шага, на котором он остановился.

## 🚀 Детальное описание ключевых методов класса `Game`

- **`init()`** — базовая инициализация приложения: загрузка данных из хранилища, рендер базового UI и биндинг событий. Если игра была прервана на этапе ожидания ввода, вызывается повторное воспроизведение последовательности `playSequence()`.
- **`startRound()`** и **`nextRound()`** — методы контроля раунда. В неблокирующем режиме через `await` ожидают отсчёт `RSG()` (Ready, Set, Go!), демонстрируют игроку текущую комбинацию, и только затем переключают стейт в `WAITING_INPUT`.
- **`handleBlocks()`** — слой валидации пользовательского ввода. Строго сверяет клик пользователя с `sequence[this.#currentStep]`. При совпадении — сдвигает внутренний указатель шага или начинает новый раунд. При ошибке — переводит машину состояний в `GAME_OVER` и синхронизирует рекорд.
- **`renderCards()`** и **`renderHUD()`** — чистые абстракции над DOM API для предсказуемой и динамической отрисовки элементов без жесткой привязки к HTML.

Данная реализация показывает глубокое понимание контекста выполнения (`this`), асинхронности в JavaScript, работы событийного цикла (Event Loop) и современных паттернов написания безопасного и масштабируемого кода на уровне Advanced.
