# Dynamic Post Feed (JavaScript Semester 2)

## 📌 О проекте

Данный проект представляет собой динамическую ленту постов (Feed), разработанную с использованием нативного JavaScript (
Vanilla JS). Основная цель проекта — демонстрация профессиональных навыков работы с асинхронным JavaScript, паттернами
проектирования (сильная связность и слабое зацепление), а также глубокого понимания взаимодействия с DOM и Web API.

Приложение реализует паттерн **MVC-подобной архитектуры**, разделяя ответственность между различными модулями (API,
State Management, UI и Controller), что обеспечивает высокую масштабируемость и легкость поддержки кодовой базы.

## ⚙️ Основной функционал

- **Асинхронная загрузка данных (Fetch API):** Интеграция с внешним REST API (`jsonplaceholder.typicode.com`) для
  получения постов с поддержкой пагинации (`_page`, `_limit`) и серверного поиска (`q`).
- **Бесконечный скролл (Infinite Scrolling):** Использование современного API браузера `IntersectionObserver` для
  автоматической подгрузки постов при достижении пользователем конца списка, без необходимости ручного нажатия кнопок.
- **Интеллектуальный поиск с Debounce (Оптимизация):** Поиск по постам выполняется "налету", но для предотвращения
  избыточных сетевых запросов (перегрузки сервера) применен паттерн **Debounce** (задержка в 500 мс после последнего
  ввода).
- **Управление состояниями (State Management):** Выделенный модуль менеджера состояний гарантирует консистентность
  данных (отслеживание текущей страницы, флагов загрузки, наличия дополнительных данных для загрузки).
- **Graceful Error Handling:** Перехват и корректная обработка сетевых ошибок, а также отображение соответствующих
  UI-уведомлений (состояния "ничего не найдено", лоадеры, сообщения об ошибках).

## 🗂 Архитектура и Структура файлов

Проект построен с акцентом на объектно-ориентированный JavaScript (ES6 Classes) и инкапсулирует логику в профильных
классах.

- **`index.html`** — Семантический каркас страницы. Реализует структуру списка, поля ввода, элементы для лоадеров и "
  наблюдателя" (`sentinel`), необходимого для работы `IntersectionObserver`.
- **`style.css`** — Стилизация приложения.
- **`api.js` (Класс `Api`)** — Слой абстракции данных (Data Access Layer). Изолирует всю логику формирования
  HTTP-запросов. Использует `URL` и `URLSearchParams` для безопасного построения параметров запроса.
- **`stateManager.js` (Класс `StateManager`)** — Провайдер состояния. Централизованно хранит переменные: `currentPage`,
  `limit`, `isLoading`, `hasMore`, `searchQuery`. Обеспечивает единый источник истины (Single Source of Truth) для всего
  приложения.
- **`ui.js` (Класс `UI`)** — Слой представления (View). Абстрагирует всю работу с DOM. Отвечает за рендеринг элементов (
  через `createElement` для безопасности от XSS), очистку списков, управление классами видимости (`hidden`) и блокировку
  кнопок.
- **`app.js` (Класс `App`)** — Контроллер приложения. Является связующим звеном. Инициализирует зависимости (`Api`,
  `UI`, `StateManager`), привязывает слушатели событий (Event Listeners) и координирует потоки данных.

## 🧠 Подробный разбор ключевых механизмов

### 1. Механизм бесконечной ленты (IntersectionObserver)

Вместо классического (и неоптимизированного) отслеживания события `scroll` на объекте `window`, приложение использует
`IntersectionObserver`.
В классе `App` (метод `setObserver()`) создается наблюдатель, который следит за невидимым элементом `#sentinel` в самом
низу списка. Когда `#sentinel` попадает в область видимости `isIntersecting === true`, контроллер проверяет состояние
`!isLoading` и `hasMore`, после чего инициирует загрузку следующей страницы (`loadPosts('append')`). Это значительно
эффективнее с точки зрения производительности (FPS) браузера.

### 2. Оптимизация сети (Debounce)

При вводе текста в поле `#searchInput` срабатывает событие `input`. Чтобы не отправлять GET-запрос при вводе каждой
буквы, используется таймер (`setTimeout`). Каждый новый ввод сбрасывает предыдущий таймер (
`clearTimeout(this.debounce)`). Запрос уйдет только в том случае, если пользователь прекратил печатать на 500
миллисекунд.

### 3. Режимы рендеринга (`render` vs `append`)

Функция `loadPosts(mode)` поддерживает два параметра:

- `render` — применяется при инициализации приложения или при новом поиске. Очищает текущий список и отрисовывает
  результаты с первой страницы.
- `append` — применяется при скролле или нажатии "Загрузить ещё". Добавляет новые DOM-узлы в конец существующего списка
  без перерисовки старых элементов (оптимизация работы с DOM tree).

## 🚀 Как запустить проект

Поскольку проект написан на чистом JS без использования сборщиков (бандлеров) типа Webpack или Vite, запуск максимально
прост:

1. Откройте файл `index.html` в любом современном браузере.
2. Альтернативно, для избежания проблем с CORS (если бы запросы шли на локальные файлы) и для симуляции реального
   окружения, рекомендуется использовать плагин **Live Server** (в VS Code / WebStorm) или любой локальный HTTP-сервер.

## 🎓 Резюме

Архитектурный подход в данном приложении демонстрирует зрелое понимание принципов проектирования клиентских
веб-приложений. Разделение кода на слои (Data, State, UI, Controller) делает код читаемым, устойчивым к ошибкам и легко
расширяемым. Замена `window.onscroll` на `IntersectionObserver` и внедрение `Debounce` подчёркивают фокус на
производительности и пользовательском опыте (UX).
